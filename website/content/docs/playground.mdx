# Interactive Playground

Explore Pothos GraphQL schema building with interactive examples. Each example shows the TypeScript code, generated GraphQL schema, and lets you run queries.

## Usage Patterns

You can embed playground-enabled code blocks using different patterns:

### 1. Inline Code with Playground

Add the `playground` attribute to any code block to enable the "Open in Playground" button. The code from the block will be loaded when clicked:

```ts playground
import SchemaBuilder from '@pothos/core';

const builder = new SchemaBuilder({});

builder.queryType({
  fields: (t) => ({
    hello: t.string({
      args: {
        name: t.arg.string({ required: false }),
      },
      resolve: (_, args) => `Hello, ${args.name ?? 'World'}!`,
    }),
  }),
});

export const schema = builder.toSchema();
```

### 2. With Pre-Registered Examples

Reference a pre-registered example by ID. The example's full configuration (multiple files, default query) will be loaded:

```ts playground example="getting-started-first-schema"
import SchemaBuilder from '@pothos/core';

const builder = new SchemaBuilder({});

// This code is shown in the docs, but when opened in playground
// it loads the full "getting-started-first-schema" example with all its files.
const Character = builder.objectRef<ICharacter>('Character');

Character.implement({
  fields: (t) => ({
    name: t.exposeString('name'),
  }),
});
```

### 3. With GraphQL Query

Provide a query to pre-populate the GraphiQL tab and automatically switch to it:

```ts playground query="{ hello }"
import SchemaBuilder from '@pothos/core';

const builder = new SchemaBuilder({});

builder.queryType({
  fields: (t) => ({
    hello: t.string({
      resolve: () => 'Hello, World!',
    }),
  }),
});

export const schema = builder.toSchema();
```

### 4. Example + Custom Query

Combine an example with a custom query to guide users to specific functionality:

```ts playground example="getting-started-first-schema" query="{ frodo { id name } }"
// Opens the getting-started-first-schema example with a pre-populated query
```

### 5. Switchable Definition Styles

To show the same schema implemented several ways — e.g. object refs vs. classes vs. builder types — give each fence a `tab` label. Consecutive fences that share the `tab` attribute merge into a single block with a style-switcher, and "Open in Playground" opens whichever tab is selected:

````mdx
```ts playground example="fundamentals-objects" tab="Object refs"
// primary style — the default (first) tab
```

```ts playground example="fundamentals-objects-variant-classes" tab="Classes"
// alternative style
```

```ts playground example="fundamentals-objects-variant-builder-types" tab="Builder types"
// alternative style
```
````

The `tab` attribute drives the switcher (via fumadocs' built-in code-block tabs). Each fence's `example` is the exact bundle it opens: the base id for the default style, and `<base>-variant-<slug>` for the others. Author the alternatives as `variant-<slug>/` subdirectories of the base bundle (see the [playground examples README](https://github.com/hayes/pothos/blob/main/website/playground-examples/README.md)). A bundle uses steps **or** variants, never both.

## How It Works

Each playground has three tabs:

1. **Code** - The TypeScript source code using Pothos
2. **Schema** - The generated GraphQL SDL
3. **GraphiQL** - An interactive query editor

Try modifying the queries in the GraphiQL tab to explore the schema!
