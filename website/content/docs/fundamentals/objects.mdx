Object types are the bulk of any GraphQL schema. In Pothos, you declare them with `builder.objectRef` and fill in the fields with `implement`. The same type can be defined a few ways — switch between them below; `objectRef` is the one to reach for first.

The generic argument to `objectRef` — `IRace` here — is the *backing model*. That's the TypeScript shape your resolvers will return and Pothos will hand back to you as `parent` on any field defined under this type. The rest of this page uses the `objectRef` style.

## Why objectRef

Splitting the declaration (`objectRef`) from the definition (`implement`) means two types can reference each other without forward-declaration gymnastics:

```typescript
const Character = builder.objectRef<ICharacter>('Character');
const Faction = builder.objectRef<IFaction>('Faction');

Character.implement({
  fields: (t) => ({
    factions: t.field({ type: [Faction], resolve: (c) => c.factions }),
  }),
});

Faction.implement({
  fields: (t) => ({
    members: t.field({ type: [Character], resolve: (f) => f.members }),
  }),
});
```

`objectRef` is also the only way to attach a backing model without registering it on the `SchemaTypes` generic. For most projects that's all you need.

## Returning the type

Once a `Race` reference exists you can use it as the type of any field, anywhere:

```typescript
builder.queryType({
  fields: (t) => ({
    race: t.field({
      type: Race,
      args: { id: t.arg.id({ required: true }) },
      resolve: (_root, { id }) => findRaceById(String(id)),
    }),
  }),
});
```

The resolver must return something that satisfies `IRace`. If your data source already produces that shape (database row, ORM model, in-memory record) you usually return it untouched.

## Other styles

The **Classes** and **Builder types** tabs on the first example are the same `Race` schema in Pothos' two older styles. Reach for them only when you need the specific affordance:

- **Class-backed types.** `builder.objectType(MyClass, { ... })` infers the backing model from a TypeScript class. Useful if you already keep domain classes around for behavior.
- **SchemaTypes-named types.** Register the type on the `Objects` generic, then refer to it by name as a string: `t.field({ type: 'Race', ... })`. The Prisma and Drizzle plugins lean on this so they can wire type names to ORM model names.

Both interoperate with `objectRef` — pick whichever fits a given type and mix freely.
