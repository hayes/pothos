The Query root is the entry point for every read your clients perform. Pothos offers three ways to attach fields to it; you can mix freely within one schema.

<include cwd lang="typescript" meta='playground example="fundamentals-queries"'>playground-examples/fundamentals-queries/schema.ts#query-type</include>

`queryType` runs exactly once per schema — it creates the `Query` type itself. Even if you plan to add fields piecewise later, call `queryType({})` once to bring `Query` into existence.

## queryField for one at a time

To keep query fields next to the type they return, use `queryField`:

<include cwd lang="typescript">playground-examples/fundamentals-queries/schema.ts#query-field</include>

This is the most useful pattern in a modular schema: each module owns its type *and* the query entrypoints that return it. See [Project layout](../patterns/project-layout) for the full setup.

## queryFields for a handful in one place

When several queries logically belong together, `queryFields` saves the repeated boilerplate:

```typescript
builder.queryFields((t) => ({
  races: t.field({
    type: [Race],
    resolve: () => Races,
  }),
  raceCount: t.int({
    resolve: () => Races.length,
  }),
}));
```

The shape inside the function is the same `fields:` map you'd pass to `queryType`.

## When to use which

- **`queryType` once.** Always, to declare the Query root. Optionally pass an initial `fields:` map if your schema is small enough that one place is the right place.
- **`queryField` per field.** When you want one query entry next to its type definition.
- **`queryFields` per module.** When a module owns a handful of related queries and you'd rather group them than declare each individually.

All three coexist in one schema. The choice is about file organization, not behavior.
