Start here if you're new to Pothos. Three short pages take you from an empty project to a running GraphQL endpoint:

- **[Introduction](./introduction)** — what Pothos is, how it differs from schema-first approaches, the mental model the rest of the docs assume.
- **[Installation](./installation)** — install `@pothos/core`, configure `tsconfig.json`, and write a first schema.
- **[First server](./first-server)** — wire the schema into `graphql-yoga` and run a query against it.

After that, [Fundamentals](../fundamentals) is the per-concept reference and [Patterns](../patterns) covers the larger shapes that recur across applications.

<Callout type="info">
  Already have Pothos installed? Skip to [SchemaBuilder](../fundamentals/schema-builder) for the per-concept reference.
</Callout>
