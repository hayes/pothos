This is the per-concept reference. Each page covers one piece of the Pothos API in enough depth to use it well, no further.

The pages sit in dependency order: each one only refers backward, not forward. If you read top-to-bottom you'll never hit a concept the docs haven't introduced yet. But every page also stands alone — drop in at whichever one you need.

The reference splits naturally into three layers:

- **Setup.** [SchemaBuilder](./schema-builder) — the generic argument and the constructor options that match it.
- **Defining types and fields.** [Object types](./objects), [Fields](./fields), [Resolvers](./resolvers), [Context](./context), [Arguments](./args), [Input objects](./inputs).
- **Root operations.** [Queries](./queries), [Mutations](./mutations), [Subscriptions](./subscriptions).
- **Polymorphism and custom types.** [Interfaces](./interfaces), [Unions](./unions), [Enums](./enums), [Scalars](./scalars).
