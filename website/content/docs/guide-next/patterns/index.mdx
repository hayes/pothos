These pages cover the shapes that recur across applications — bigger than a single field, smaller than a full architecture. Each one stands alone; read them as you need them, not in order.

- **[Project layout](./project-layout)** — how to grow a single-file schema into something modular without making the build harder to reason about.
- **[Handling errors](./handling-errors)** — throwing from resolvers, masking in production, and when to graduate to typed error unions.
- **[Default nullability](./default-nullability)** — flipping Pothos's nullable-by-default to non-nullable, when that's worth it.
- **[Circular references](./circular-references)** — two types that point at each other without forward-declaration pain.
- **[Inferring types](./inferring-types)** — pulling TypeScript types back out of your Pothos refs.
- **[Printing and codegen](./printing-and-codegen)** — emitting an SDL string and generating client types.
- **[Reusable fields](./reusable-fields)** — sharing field definitions across types.

<Callout type="info">
  Each pattern picks one approach and shows it end to end. Where alternatives exist, they live in a "Other styles" disclosure near the end of the page.
</Callout>
