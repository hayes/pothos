Pothos builds a GraphQL schema from TypeScript code. You define types, fields, and resolvers as values; Pothos infers the types and hands you a `GraphQLSchema` for any server. There are no schema files to parse, no decorators, and no code-generation step — the schema is a regular TypeScript module, so your editor's go-to-definition and refactors work across it.

The core adds zero runtime overhead and depends only on `graphql`. Everything else — auth, errors, validation, Relay, Prisma, Drizzle — arrives as a plugin that adds methods on the builder and composes with the rest.

Open the example in the playground to run it, edit the resolver, and watch the SDL update. For the full narrative — how Pothos differs from schema-first tools, what it does and doesn't do, and when to reach for it — read the [Introduction](./getting-started/introduction).

## What sets Pothos apart

- **Type-safe without codegen or decorators.** Type inference flows through every resolver; `parent`, `args`, and `ctx` arrive fully typed, and a field can't return a value that doesn't fit its GraphQL type.
- **A clear split between API and data.** The GraphQL shape clients see stays separate from the backing model your resolvers return.
- **A cohesive plugin ecosystem.** Plugins interoperate rather than collide — each feels like it was built into the core.
- **Proven at scale.** Pothos runs from small prototypes to large production schemas, and is in use at companies including Airbnb and Netflix.
